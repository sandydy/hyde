HTML Autocomplete

I used to create html from plain text. But autocomplete feature is nice which increases productivity. Notepad++ does have autocomplete capability but by default is disabled. After googling a bit. Here is the setting

Settings->Preferences->Backup/AutoCompletion->Auto-Completion

You can check the options, then when you go back to html file, start typing, the suggestions will come up. Same applies to javascript, but if you are looking for the feature like Intellisense in Visual Studio, this is still far from enough. e.g. when you type "win", the editor will suggest you "window", but you put a period there, nothing shows up...

HTML auto-endtag

when you type <div> surely you want to end the tag by </div> but you will need to install a plugin TextFx for this feature. To install the plug-in, 

Plugins->Plugin Manager->Show Plugin Manager

in the dialog box, choose "TextFx" and click "Install" after installation is done, you will need to restart notepad++. Just do it. Then you will need to enable the feature by 

TextFx->TextFx Settings->Auto Close XHTML/HTML Tag

There are also other settings to auto close brackets, you can try it yourself.

JSLint

Here comes to the topic I want to try today, JSLint, which is a tool to scan your javascript and suggest any potential problems or improvements. It is like Fxcop for .NET. Again, there is a plug-in called JSLint, simply follow instruction above to locate and install the plug-in. Then open a js file and execute the scan via hotkey, Ctrl+Shift+F5. You will then see a list of items at the JSLint sub-window. it is nice and easy to use. 

There is a package called JSHint, which is forked from JSLint with the same creator. JSHint integrates with node JS and provide CLI application to scan via the code. I will leave this topic to later post.

In any case, for small application or single developer application, the notepad++ plugin will be enough. For larger scale projects, I will need to explore more on how to integrate JSLint/JSHint to build process (surely js doesn't require compilation, For example, if a project involves a few developers, every time a developer commit a change to svn and the JSLint will be triggered and generate a report, that will be nice. This is what we've done for .NET application, in web application, we are obviously expecting same thing. wait for my later post to give update on this.
